# Climate Monitor (Django)

Εφαρμογή Django που δέχεται σε πραγματικό χρόνο μετρήσεις θερμοκρασίας/υγρασίας από
πολλαπλές συσκευές, τις συγκρίνει με τον πίνακα εργασίας/ανάπαυσης «Μέτρια εργασία»
(32-45°C, ανά βαθμό Κελσίου) και στέλνει **SMS** όταν χρειάζεται διάλειμμα ή διακοπή
εργασίας. Η αρχική σελίδα δείχνει γράφημα θερμοκρασίας/υγρασίας σε πραγματικό χρόνο, και
κάθε συσκευή αλλάζει χρώμα (πράσινο → κίτρινο → κόκκινο) ανάλογα με το επίπεδο κινδύνου.

## 1. Εγκατάσταση

```bash
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
pip install -r requirements.txt

copy .env.example .env        # Windows  (Linux/Mac: cp .env.example .env)
# άνοιξε το .env και συμπλήρωσε τα στοιχεία SMS gateway σου (προαιρετικό, μόνο για SMS - βλ. §5)
# Το .env διαβάζεται ΑΥΤΟΜΑΤΑ - δεν χρειάζεται κανένα "export" σε καμία πλατφόρμα.

python manage.py migrate
python manage.py createsuperuser
python manage.py runserver
```

Άνοιξε:
- **http://127.0.0.1:8000/** — η αρχική οθόνη: συσκευές, τρέχον επίπεδο, γράφημα, και κουμπί **🔑 Admin** πάνω-δεξιά
- **http://127.0.0.1:8000/thresholds/** — πίνακας θερμοκρασίας/υγρασίας + ρυθμίσεις επιπέδων
- **http://127.0.0.1:8000/admin/** — μόνο δύο πράγματα: **Devices** και **Recipients**

> **Ενημερώνεις υπάρχουσα εγκατάσταση;** Αντικατέστησε όλα τα αρχεία **εκτός** από το
> δικό σου `db.sqlite3` (μην το σβήσεις — έχει τις συσκευές/παραλήπτες σου), μετά τρέξε
> `python manage.py migrate` για να προστεθεί το νέο πεδίο σύνδεσης
> συσκευών↔παραληπτών. Δεν χρειάζεται `createsuperuser` ξανά.

## 2. Σύνδεση μέσω Ecowitt Gateway (πραγματικός χρόνος, καμία εγγραφή/κόστος)

Αν χρησιμοποιείς **Ecowitt** αισθητήρα (WN32/WH32 ή πολυκάναλο WH31) + **Gateway**
(GW1100/GW1000/GW2000), η σύνδεση είναι αυτόματη και σε πραγματικό χρόνο — το Gateway
στέλνει δεδομένα απευθείας σε αυτόν τον server, χωρίς κανένα ενδιάμεσο cloud.

**Α) Στη συσκευή (εφαρμογή WS View Plus / Ecowitt):**
1. Άνοιξε **Device List → [το Gateway σου] → Weather Services**.
2. Βρες την επιλογή **"Customized"** και ενεργοποίησέ την.
3. Συμπλήρωσε:
   - **Protocol Type**: `Ecowitt`
   - **Server IP/Hostname**: η IP/domain του server μας (π.χ. `192.168.1.50` αν είναι
     στο ίδιο δίκτυο, ή δημόσιο domain αν είναι online)
   - **Path**: `/api/ecowitt/`
   - **Port**: όποιο port τρέχει ο server μας (π.χ. `8000` για `runserver`)
   - **Upload Interval**: πρότεινε 60-300 δευτερόλεπτα
4. Πάτα Save. Σημείωσε το **PASSKEY** που δείχνει η οθόνη του Gateway (το χρειάζεσαι
   στο επόμενο βήμα).

**Β) Στο δικό μας admin (`/admin/monitor/device/`):**
1. **Add Device**, γράψε ένα όνομα.
2. Στην ενότητα **"Ή σύνδεση μέσω Ecowitt Gateway"**, βάλε το **PASSKEY** που σημείωσες.
3. Άφησε το **Channel** κενό αν έχεις τον απλό αισθητήρα (WN32/WH32). Αν έχεις
   πολυκάναλο **WH31**, βάλε εδώ τον αριθμό καναλιού (1-8) — και πρόσθεσε **ξεχωριστή
   Device εγγραφή για κάθε κανάλι/τοποθεσία** (όλες με το ίδιο PASSKEY, διαφορετικό
   Channel).
4. Save. Μέσα σε λίγα λεπτά (ανάλογα το Upload Interval που όρισες) θα δεις τη
   συσκευή να ενημερώνεται στην αρχική σελίδα.

Η θερμοκρασία που στέλνει το Ecowitt είναι σε Fahrenheit — μετατρέπεται αυτόματα σε
Celsius πριν αποθηκευτεί, δεν χρειάζεται καμία ρύθμιση από τη μεριά σου.

> **⚠ Σημαντική επιφύλαξη**: έφτιαξα αυτό το endpoint βασισμένος στην τεκμηρίωση του
> πρωτοκόλλου Ecowitt (πεδία `PASSKEY`, `tempf`, `humidity`, `temp1f`...`temp8f`,
> `humidity1`...`humidity8`, `dateutc`) — **δεν το δοκίμασα με πραγματική συσκευή**,
> αφού δεν έχω μία στη διάθεσή μου. Αν το πρώτο μήνυμα δεν αναγνωριστεί σωστά, **δες την
> κονσόλα του `runserver`**: κάθε φορά που έρχεται μήνυμα με άγνωστο PASSKEY ή με
> κανάλι που δεν ταιριάζει, καταγράφεται εκεί ολόκληρο το μήνυμα που ελήφθη ώστε να
> δούμε μαζί τα πραγματικά ονόματα πεδίων και να διορθώσουμε αν χρειαστεί.

## 3. ⚠ Σχετικά με τον αισθητήρα Lascar EL-WiFi-TH (αν τον έχεις ήδη)

Αν χρησιμοποιείς Lascar EL-WiFi-TH (ή παρόμοιο EasyLog προϊόν): αυτή η συσκευή είναι
κλειδωμένη να στέλνει δεδομένα **μόνο** στους servers της Lascar (easylogcloud.com),
όχι σε δικό μας server — δεν υπάρχει δημόσιο API να τραβήξουμε δεδομένα αυτόματα από
εκεί. Η μόνη λύση εκεί είναι χειροκίνητο import CSV:

1. Μπες στο **easylogcloud.com**, άνοιξε τη συσκευή σου, **Data → Export**.
2. Στο `/admin/monitor/device/`, βρες τη συσκευή σου και πάτα **«📁 Upload CSV»**.
3. Ανέβασε το αρχείο. Ο parser αναγνωρίζει αυτόματα τις στήλες θερμοκρασίας/υγρασίας/
   ώρας ανεξάρτητα από τα ακριβή ονόματα.
4. Κάθε γραμμή αξιολογείται στον πίνακα ορίων σου. SMS στέλνεται **μόνο** για την πιο
   πρόσφατη μέτρηση του αρχείου.
5. Εναλλακτικά από τερματικό: `python manage.py import_csv <device_id> αρχείο.csv`

Στο φάκελο `sample_data/example_easylog_export.csv` υπάρχει ένα δοκιμαστικό (mock)
αρχείο. **Περιορισμός**: αυτό είναι χειροκίνητο, όχι πραγματικού χρόνου — για αυτό
προτείνουμε τα Ecowitt (§2) ή DIY ESP32 αντί για Lascar.

## 4. Βήμα-βήμα: πώς συνδέεις μια συσκευή μέσω του δικού μας JSON API (π.χ. DIY ESP32)

1. Πήγαινε στο `/admin/monitor/device/`, πάτα **«Add Device»**, γράψε ένα όνομα (π.χ.
   "Δοκιμή"), πάτα **Save**.
2. Θα ανοίξει ξανά η σελίδα της συσκευής — θα δεις ένα `api_key` και από κάτω ένα έτοιμο
   παράδειγμα `curl` με το πραγματικό κλειδί.
3. Άνοιξε ένα δεύτερο τερματικό (χωρίς να κλείσεις το `runserver`) και είτε:
   - κάνε τεστ με πραγματικά δεδομένα (αντίγραψε-επικόλλησε το `curl` από το admin), είτε
   - τρέξε τον προσομοιωτή: `python manage.py simulate_device --interval 10`
4. Άνοιξε (ή ανανέωσε) το `http://127.0.0.1:8000/` — η συσκευή θα εμφανιστεί μόνη της
   σαν καρτέλα μέσα σε λίγα δευτερόλεπτα (η λίστα συσκευών ανανεώνεται αυτόματα, δεν
   χρειάζεται πια F5).
5. Αν δεν έχεις ορίσει ακόμα πίνακα ορίων στο `/thresholds/`, οι μετρήσεις θα
   αποθηκεύονται κανονικά αλλά θα λένε **"Χωρίς πίνακα ορίων ακόμα"** αντί για επίπεδο —
   φτιάξε πρώτα τον πίνακα (βλ. §5) για να ενεργοποιηθούν τα επίπεδα/SMS.

**Σημαντικό**: δεν μπόρεσα να τρέξω πραγματικά τον server σε αυτό το περιβάλλον (δεν έχω
πρόσβαση σε internet εδώ για `pip install`), οπότε δεν το δοκίμασα uçtan-uça. Αν κάτι δεν
συμπεριφέρεται όπως περιγράφω, στείλε μου το ακριβές μήνυμα/σφάλμα και το διορθώνουμε.

## 5. Ο πίνακας θερμοκρασίας/υγρασίας (`/thresholds/`)

- Πάτα **«Φόρτωση πίνακα Μέτρια εργασία (32-45°C)»** για να γεμίσει αμέσως ο πίνακας με
  τα ακριβή νούμερα από το φύλλο αναφοράς σου. Μετά μπορείς να αλλάξεις, να προσθέσεις ή
  να διαγράψεις οποιαδήποτε γραμμή θερμοκρασίας.
- Κάθε γραμμή έχει 4 αριθμούς: μέχρι ποιο % υγρασίας ισχύει το Επίπεδο 1, 2, 3, 4. Πάνω
  από τον 4ο αριθμό είναι αυτόματα Επίπεδο 5.
- Θερμοκρασία κάτω από τη μικρότερη γραμμή → Επίπεδο 1 (ασφαλές). Πάνω από τη μεγαλύτερη
  γραμμή → Επίπεδο 5 (επικίνδυνο), ό,τι κι αν δείχνει η υγρασία.
- Στο κάτω μέρος της σελίδας υπάρχουν **5 κάρτες** (μία ανά επίπεδο 1-5), όπου ορίζεις
  **μία φορά** το πρόγραμμα διαλείμματος, το κείμενο SMS, και αν στέλνεται ειδοποίηση.
  Δεν χρειάζεται όνομα ή σειρά — αυτά καθορίζονται αυτόματα από το επίπεδο (1-5).

Οι αλλαγές ισχύουν αμέσως, χωρίς restart του server.

## 6. Ειδοποιήσεις SMS

Στο `/admin/monitor/recipient/` πάτα **«Add Recipient»**:
- **Name**: για αναγνώριση (π.χ. "Γιώργος - Επόπτης").
- **Phone number**: διεθνής μορφή, π.χ. `+35799123456`.
- **Receive sms**: άφησέ το τικαρισμένο.
- **Devices**: επίλεξε ρητά για ποιες συσκευές θα ειδοποιείται (μπορείς να διαλέξεις
  πολλές). **Χωρίς καμία επιλεγμένη συσκευή, αυτό το άτομο ΔΕΝ θα λάβει κανένα SMS.**
  Η ίδια συσκευή μπορεί να έχει πολλούς παραλήπτες, και ο ίδιος παραλήπτης μπορεί να
  ακούει πολλές συσκευές.

Χρειάζεσαι το **[SMSGate](https://sms-gate.app/)** — ανοιχτού κώδικα, χωρίς εγγραφή,
χρησιμοποιεί το δικό σου Android κινητό/SIM ως πύλη SMS, ουσιαστικά χωρίς όριο μηνυμάτων
(μόνο ό,τι επιτρέπει το δικό σου πακέτο SMS):

1. Εγκατέστησε την εφαρμογή **SMSGate** στο Android κινητό σου.
2. Άνοιξέ την, ενεργοποίησε τον διακόπτη **"Cloud Server"**, πάτα **"Online"** για να συνδεθεί.
3. Θα εμφανιστούν αυτόματα ένα **Username** και **Password** (δεν χρειάζεται καμία εγγραφή) -
   σημείωσέ τα.
4. Βάλε τα στοιχεία στο `.env`:

```
SMS_GATEWAY_URL=https://api.sms-gate.app
SMS_GATEWAY_USERNAME=το_username_που_σου_έδειξε_το_app
SMS_GATEWAY_PASSWORD=το_password_που_σου_έδειξε_το_app
```

(Υπάρχει και **Local mode** αν προτιμάς ο server σου να μιλάει απευθείας στο κινητό μέσα στο
ίδιο δίκτυο, χωρίς καθόλου cloud - τότε το `SMS_GATEWAY_URL` γίνεται κάτι σαν
`http://192.168.1.50:8080`.)

Δεν στέλνεται SMS σε κάθε μέτρηση — μόνο όταν αλλάζει επίπεδο ή όταν περάσουν
`NOTIFICATION_COOLDOWN_MINUTES` (προεπιλογή 15) από την τελευταία ειδοποίηση ίδιου
επιπέδου.

**Για να επιβεβαιώσεις ότι η αποστολή SMS δουλεύει**, χωρίς να περιμένεις πραγματική
υπέρβαση ορίου:
- Στη λίστα `/admin/monitor/recipient/`, πάτα **«📩 Send test SMS»** δίπλα σε έναν
  παραλήπτη — στέλνει αμέσως ένα δοκιμαστικό μήνυμα.
- Στη σελίδα κάθε συσκευής (`/admin/monitor/device/<id>/change/`), κάτω-κάτω υπάρχει
  **«Ιστορικό ειδοποιήσεων SMS»** — δείχνει τις τελευταίες 10 προσπάθειες αποστολής για
  αυτή τη συσκευή, με ✓/✗ και τυχόν μήνυμα σφάλματος.

## Logs (καταγραφή) και μακροπρόθεσμη διαχείριση δεδομένων

Η εφαρμογή καταγράφει τι κάνει σε δύο μέρη ταυτόχρονα:
- **Στην κονσόλα** που τρέχει το `runserver` (ζωντανά, βλέπεις τα πάντα εκεί την ώρα που συμβαίνουν).
- **Σε αρχείο**, στο `logs/app.log` μέσα στον φάκελο του project. Άνοιξέ το με οποιοδήποτε
  πρόγραμμα κειμένου (Notepad, VSCode), ή για να το παρακολουθείς ζωντανά σε PowerShell:
  ```powershell
  Get-Content logs\app.log -Wait -Tail 20
  ```
  Το αρχείο γυρίζει αυτόματα (rotation) στα 5MB - κρατάει τα τελευταία 5 αρχεία (μέχρι
  ~25MB συνολικά), δεν γεμίζει επ' άπειρον τον δίσκο σου.

**Θα γεμίσει/κρασάρει η βάση με τον καιρό;** Όχι σύντομα - ακόμα και με πολλές συσκευές
για χρόνια, μιλάμε για μερικά εκατοντάδες MB, απόλυτα εντός των δυνατοτήτων του SQLite.
Αν κάποια στιγμή θες να "καθαρίσεις" πολύ παλιά ιστορικά δεδομένα (προαιρετικό):
```powershell
python manage.py cleanup_old_readings --days 365 --dry-run   # δες πόσα ΘΑ διαγράφονταν, χωρίς να διαγραφεί τίποτα
python manage.py cleanup_old_readings --days 365             # πραγματική διαγραφή
```

## Αντιμετώπιση προβλημάτων

**"no such table"**: δεν έτρεξες `python manage.py migrate`.

**Η συσκευή δεν εμφανίζεται στην αρχική**: περίμενε λίγα δευτερόλεπτα (η λίστα
ανανεώνεται αυτόματα) ή κάνε F5. Αν συνεχίσει να μην εμφανίζεται, έλεγξε ότι η συσκευή
είναι **is_active** στο admin.

**Οι μετρήσεις έρχονται αλλά δεν δείχνει επίπεδο**: δεν έχεις φτιάξει ακόμα τον πίνακα
στο `/thresholds/` — πάτα το κουμπί «Φόρτωση πίνακα Μέτρια εργασία».

**Δεν στέλνονται SMS**: έλεγξε ότι έχεις συμπληρώσει τα `SMS_GATEWAY_*` στο `.env`, ότι ο
παραλήπτης έχει `receive_sms` τικαρισμένο, και ότι το αντίστοιχο Επίπεδο στο
`/thresholds/` έχει «Αποστολή SMS» τικαρισμένο.
